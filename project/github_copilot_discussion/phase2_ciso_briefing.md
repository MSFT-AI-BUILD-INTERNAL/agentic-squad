## 1. 위협 모델 분석

| 위협 시나리오 | 공격 벡터 | 발생 가능성 | 영향도 |
|---|---|---|---|
| **코드 자산 외부 유출** | Copilot 프롬프트를 통한 민감 코드 컨텍스트 전송 | 중 | 상 |
| **시크릿 노출** | AI 생성 코드에 하드코딩된 자격증명 포함 | 중 | 상 |
| **공급망 공격** | Copilot 추천 의존성에 악성 패키지 포함 | 중 | 상 |
| **Agent 권한 남용** | Coding Agent가 의도치 않은 인프라 코드 변경 | 중 | 중 |
| **프롬프트 인젝션** | 악의적 코드 주석을 통한 AI 출력 조작 | 하 | 중 |
| **내부자 위협** | Copilot을 이용한 보안 통제 우회 코드 생성 | 하 | 상 |

**핵심 판단**: 대부분의 위협은 기존 SDLC 보안 통제로 완화 가능하며, GitHub 제공 도구가 주요 벡터를 커버함.

## 2. 코드 자산 보호 평가

| 항목 | 평가 | 비고 |
|---|---|---|
| **모델 학습 제외** | ✅ 충분 | Enterprise 플랜은 고객 코드 학습 제외가 기본 정책. 계약상 명시 확인 필요 |
| **전송 암호화** | ✅ 충분 | TLS 1.2+ 적용 |
| **저장 암호화** | ✅ 충분 | AES-256, 업계 표준 부합 |
| **Data Residency** | ⚠️ 조건부 | 현재 미국 기반. 국내 데이터 주권 요구 시 계약 조항으로 보완 필요 |
| **프롬프트 데이터 잔류** | ✅ 수용 가능 | 실시간 처리 후 폐기 정책. 단, 감사 로그 보존 기간 확인 필요 |
| **Public Code 필터** | ✅ 활성화 필수 | 라이선스 리스크 차단을 위해 조직 정책으로 강제 |

**판정**: 모델 학습 제외 + 암호화 수준은 수용 가능. Data Residency는 계약 보완으로 관리.

## 3. 규제 컴플라이언스 분석

| 규제/인증 | GitHub 충족 여부 | Gap |
|---|---|---|
| **ISO 27001:2022** | ✅ 인증 보유 | — |
| **SOC 2/3 Type II** | ✅ 인증 보유 | — |
| **ISO 42001:2023** (AI 거버넌스) | ✅ 인증 보유 | AI 도입 감사 시 핵심 근거로 활용 가능 |
| **개인정보보호법(PIPA)** | ⚠️ 부분 | 코드에 개인정보 포함 시 해외 전송 이슈. DPIA 수행 필요 |
| **금융권 규제(전자금융감독규정)** | ⚠️ Gap | 클라우드 이용 사전 보고, 망분리 요건 확인 필요 |
| **의료(HIPAA 등)** | ⚠️ 해당 시 | PHI 포함 코드베이스는 격리 필수 |

**핵심 Gap**: 업종별 규제(금융·의료)는 GitHub 인증만으로 불충분. 자체 통제 계층 추가 필수.

## 4. 보안 통제 체계 평가

| 도구 | 실효성 | 추가 필요 사항 |
|---|---|---|
| **Secret Scanning + Push Protection** | 🟢 높음 | 커스텀 패턴 등록(사내 토큰 형식) 필수 |
| **CodeQL (Code Scanning)** | 🟢 높음 | AI 생성 코드 전용 룰셋 추가 권장 |
| **Dependabot** | 🟢 높음 | 자동 PR 병합은 비활성화, 수동 리뷰 강제 |
| **감사 로그 + SIEM** | 🟡 보통 | 기존 SIEM(Splunk 등)과 통합 구축 필요. Copilot 사용 패턴 모니터링 대시보드 구축 |
| **조직/레포 정책 제어** | 🟢 높음 | 민감 코드 레포에 Copilot 비활성화 정책 적용 가능 |
| **Agent 통제** | 🟡 보통 | Draft PR 전용 + 인프라 코드 레포 접근 차단 설정 필요 |

## 5. 보안 가드레일 최소 요건 (도입 전 필수)

- [ ] **SAML SSO + MFA** 전사 강제 적용
- [ ] **Copilot 학습 제외** 정책 조직 수준 활성화 및 계약 명시
- [ ] **Public Code 제안 차단** 조직 정책 활성화
- [ ] **Secret Scanning Push Protection** 전체 레포 활성화 + 커스텀 패턴 등록
- [ ] **CodeQL** 기본 분석을 모든 활성 레포 CI에 통합
- [ ] **민감 코드 레포 격리**: 인프라/시크릿 관리/규제 대상 코드에 Copilot 비활성화
- [ ] **Coding Agent**: Draft PR만 허용, `main` 직접 커밋 차단, 인프라 레포 접근 금지
- [ ] **감사 로그 → SIEM 연동** 구축 완료
- [ ] **IP 허용 목록** 설정 (사내 네트워크/VPN만 접근)
- [ ] **DPIA(개인정보영향평가)** 수행 완료 (해외 데이터 전송 근거 확보)

## 6. 잔여 리스크 매트릭스

| 리스크 | 수준 | 수용 여부 | 조건 |
|---|---|---|---|
| AI 생성 코드의 미묘한 논리 결함 | 중 | ✅ 수용 | 코드 리뷰 필수화 + CodeQL 게이트 |
| 프롬프트를 통한 간접적 코드 노출 | 저 | ✅ 수용 | 학습 제외 정책 + 암호화 적용 상태 |
| Data Residency (미국 저장) | 중 | ✅ 조건부 수용 | 계약 조항 + DPIA 완료 시 |
| Agent 생성 코드의 품질 불안정 | 중 | ✅ 수용 | Draft PR + 인간 리뷰 강제 |
| 금융/의료 규제 미충족 | 고 | ❌ 미수용 | 해당 업종 코드는 파일럿 범위에서 **제외** |
| 제로데이 공급망 취약점 유입 | 저 | ✅ 수용 | Dependabot + 수동 리뷰 체계 운영 |

## 7. CISO 권고

### 결론: **조건부 도입 찬성** ✅

CTO의 단계적 접근에 동의하되, 다음 **선행 조건**을 충족한 후 파일럿을 개시해야 합니다:

1. **가드레일 최소 요건**(Section 5) 전항 충족 — 미충족 시 파일럿 불허
2. **파일럿 범위 제한**: 비규제 대상·비핵심 서비스 코드베이스로 한정
3. **보안 KPI 추가**: 파일럿 6개월간 아래 지표 모니터링
   - Secret Scanning 탐지 건수 및 평균 대응 시간
   - AI 생성 코드의 CodeQL 취약점 발견율 (기존 대비 비교)
   - Copilot 감사 로그 이상 탐지 건수
4. **전사 확대 조건**: 파일럿 기간 중 **Critical/High 보안 인시던트 0건** + 규제 감사 통과
5. **반기 재검토**: 위협 환경 변화·규제 업데이트에 따라 정책 갱신
